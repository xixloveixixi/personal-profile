# Blog Category And Tags 设计

> 创建日期：2026-06-19
> 状态：设计已确认，待用户复核

## 背景

当前博客页面直接从 Notion 数据库读取已发布文章，文章数据包含标题、状态、标签和发布日期。用户已在 Notion 数据库中新增 `选择` 字段，类型为 `Select`，选项为 `技术实践`、`AI Agent`、`项目复盘`、`学习笔记`。

本设计在不迁移 Notion 博客、不新增 Go 后端博客表的前提下，为公开博客页增加“分类 + 标签”的筛选体验。

## 目标

第一版覆盖三个目标：

1. 在博客列表页按 Notion 分类筛选文章。
2. 在分类基础上继续按 Notion 标签细筛文章。
3. 在博客列表和详情页清晰展示分类、标签和发布日期。

## 非目标

第一版明确不做：

- 不把博客迁移出 Notion。
- 不新增 MySQL 博客表、Go handler 或后台博客管理页面。
- 不新增 `/blog/category/[category]` 分类路由。
- 不把筛选状态写入 URL query string。
- 不做多选分类；每篇文章只有一个一级分类。
- 不改 RSS、sitemap 的数据源策略。

## 方案选择

采用“Notion 单选分类 + 动态标签筛选”的方案。

原因：

- 分类由 Notion 维护，符合当前博客数据源边界。
- `category` 表示内容类型，`tags` 表示具体技术点或主题词，语义清晰。
- 首版只改 Next.js 读取和展示层，范围小，验证直接。
- 后续若需要 SEO 分类页，可以在此模型上增加路由，不需要重做数据结构。

## 分类与标签语义

分类来自 Notion `选择` Select 字段，第一版固定支持：

- `技术实践`
- `AI Agent`
- `项目复盘`
- `学习笔记`

标签继续来自 Notion `标签` multi-select 字段，适合放更细颗粒度的主题，例如：

- `RAG`
- `RAGAS`
- `Next.js`
- `Go`
- `Supabase`
- `Embedding`

未设置分类的文章归入 `未分类`，文章卡片和详情页仍展示该状态。`未分类` 不作为顶部固定分类按钮，避免鼓励长期保留未整理文章。

## 数据读取

`lib/notion.ts` 增加对分类字段的读取：

- `NotionPost` 增加 `category?: string`。
- 新增 `readCategory(page)`，优先读取 `选择`，并兼容 `分类`、`Category`、`category`。
- 仅处理 Notion `select` 类型，返回 `select.name`。
- 分类缺失时返回 `undefined`，展示层再映射为 `未分类`。

`getPublishedPosts()` 仍然负责返回已发布文章列表，并继续按发布日期倒序排序。

## 列表页交互

`/blog` 页面保持 Server Component 拉取 Notion 数据，并把文章列表传给客户端筛选组件。

筛选区包含三层：

1. 搜索框：按标题、分类和标签做本地匹配。
2. 分类按钮：`全部`、`技术实践`、`AI Agent`、`项目复盘`、`学习笔记`。
3. 标签按钮：根据当前文章集合动态聚合标签。

筛选规则：

- 搜索关键词、分类和标签组合生效。
- 点击 `全部` 清空分类。
- 点击已选标签再次取消标签筛选。
- 切换分类后，如果当前选中的标签在新分类下不存在，则自动清空标签。
- 没有匹配结果时显示空状态，并提供清空筛选按钮。

标签聚合策略：

- 默认显示当前分类下出现过的标签。
- 未选分类时显示全部文章中出现过的标签。
- 标签按出现次数降序排序，同频时按字母或中文 locale 排序。

## 展示设计

文章卡片展示：

- 标题
- 分类 badge
- 发布日期
- 标签列表

文章详情页展示：

- 标题
- 分类 badge
- 发布日期
- 标签列表
- Notion 正文块

视觉上分类 badge 比标签更突出，标签保持轻量 chip，用于表示具体主题。

## 组件边界

建议新增或调整以下前端边界：

- `app/blog/page.tsx`：拉取文章数据，传给客户端组件。
- `app/blog/BlogClient.tsx`：管理搜索、分类、标签筛选状态。
- `components/blog/BlogPostCard.tsx` 或复用现有卡片：展示分类和标签。
- `app/blog/[slug]/page.tsx`：详情页展示分类。
- `lib/notion.ts`：读取 Notion 分类字段。

已有本地 MDX 博客相关的 `lib/content/blog.ts` 和旧 `BlogClient` 模式不作为本轮数据源，不引入到 Notion 列表的数据流中。

## 错误处理

- `getPublishedPosts()` 调用失败时，`/blog` 继续降级为空列表，沿用当前行为。
- 分类字段缺失或类型不匹配时，不抛错，展示为 `未分类`。
- 标签为空时不展示标签筛选行或展示空状态，不影响文章列表。
- 无 Notion 环境变量时，构建仍不应因博客页预渲染失败；博客页继续保持 `force-dynamic`。

## 验收标准

完成后应满足：

- Notion `选择` Select 字段能映射为前端文章分类。
- `/blog` 可以按分类筛选文章。
- `/blog` 可以在当前分类下按标签细筛文章。
- 搜索、分类、标签组合筛选逻辑正确。
- 文章卡片展示分类 badge、发布日期和标签。
- 文章详情页展示分类 badge、发布日期和标签。
- 未设置分类的文章显示为 `未分类`。
- `npm run build` 通过。

## 后续扩展

首版稳定后可考虑：

- 增加 `/blog?category=AI%20Agent&tag=RAG` query string，支持分享筛选结果。
- 增加 `/blog/category/[category]` 分类路由，支持 SEO 分类页。
- 将分类统计用于博客页概览。
- 在 RSS item 中补充分类字段，但不改变当前 RSS 数据源。
