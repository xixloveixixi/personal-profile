# 路由规范（NON-NEGOTIABLE）

## 文件式约定

App Router 通过 `app/` 下的目录结构组织路由，每个路由段使用 kebab-case：

| 文件 | 作用 |
|------|------|
| `page.tsx` | 路由页面入口（必填） |
| `layout.tsx` | 该层及子路由共享布局 |
| `loading.tsx` | 路由级加载态（Suspense fallback） |
| `error.tsx` | 路由级错误边界（必须 `'use client'`） |
| `not-found.tsx` | 404 边界 |
| `template.tsx` | 每次导航重新渲染的布局 |
| `route.ts` | Route Handler（仅在 `app/api/**` 内使用） |

约束：

- 禁止使用 Pages Router（`pages/` 目录）。
- 禁止建立 `Loader.tsx` + `Page.tsx` 双文件结构。
- 动态路由用 `[slug]` / `[id]`，元数据用 `generateMetadata` 函数。
- 不要在 Server Component 中使用 `useRouter`，需要导航 hook 时改为 Client Component。

## 路由权限划分（按 SDD 6.1）

| 路由 | 权限 |
|------|------|
| `/`、`/about`、`/portfolio`、`/portfolio/[slug]`、`/blog`、`/blog/[id]` | 公开 |
| `/dashboard`、`/profile`、`/goals`、`/plans`、`/progress`、`/review/weekly` | owner（需登录） |
| `/admin/*`（V1 暂不实现） | owner |

私有路由统一在 Next.js 中间件或 layout 中校验 JWT，未登录跳转登录页。

## 私有学习区共享布局（NON-NEGOTIABLE）

按 SDD 7.2，私有学习区采用「左菜单 + 右内容」工作台布局：

- 使用 `app/(dashboard)/layout.tsx` 路由组承载共享布局，通过 `components/layout/LearningWorkspaceLayout` 渲染。
- 左侧菜单固定展示：概览、学习画像、学习目标、学习计划、学习进度、周复盘。
- 右侧内容区由各路由 `page.tsx` 渲染。
- **禁止**每个私有页面自建一套页面框架。
- 移动端左菜单收起为抽屉。

## Route Handler 路径

- 公开数据：`app/api/public/<feature>/route.ts`。
- 私有数据：`app/api/private/<feature>/route.ts`。
- AI Agent 流式：`app/api/chat-agent-stream/route.ts`、`app/api/agent/*` 等保持现状。
- 详细规范见 `05-API规范.md`。
