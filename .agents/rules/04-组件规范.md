# 组件规范

## 组件分层（NON-NEGOTIABLE）

| 分层 | 目录 | 职责 |
|------|------|------|
| 原子层 | `components/ui/` | shadcn 风格原子组件（Button、Input、Dialog 等），跨业务域 |
| 通用层 | `components/shared/` | 业务无关的通用组件（ErrorBoundary、Loading、Empty 等） |
| 布局层 | `components/layout/` | 全局 Header / Footer，私有区 `LearningWorkspaceLayout` |
| 业务域 | `components/<domain>/` | 单一业务域专用组件，如 `chat`、`portfolio`、`blog`、`about` |
| 视觉层 | `components/magicui/`、`components/background/`、`components/icons/` | 视觉特效与图标 |

判定规则：

- 多业务域使用、可被任意页面调用 → 原子层 / 通用层。
- 仅服务于单一业务域 → 放入对应 `<domain>/` 子目录。
- 涉及私有学习区共享框架 → `components/layout/LearningWorkspaceLayout/`，按 SDD 7.2 实现「左菜单 + 右内容」。

**约束**：`components/ui/` 中的 shadcn 原子组件不得引入业务逻辑或 Antd 包装；业务耦合组件禁止放入。

## 组件实现规范

- 每个组件一个 PascalCase 文件（如 `ChatWindow.tsx`）；如组件较大，可建同名目录 `ChatWindow/{index.tsx, hooks.ts, types.ts}`，统一通过 `index.tsx` 暴露。
- Props 接口必须显式定义，禁止使用 `any`。
- 样式优先用 Tailwind 类；动态 class 使用 `lib/utils.ts` 中的 `cn` 工具组合。
- 交互复杂控件（表单、弹窗、Tree 等）允许使用 Antd，并基于 Antd 在 `components/<domain>/` 内做二次封装，而不是在页面层直接散用。
- 客户端交互组件首行加 `'use client'`；纯展示组件保持 Server Component。

## 组件大小与拆分

- 单个 `.tsx` 文件建议不超过 400 行，作为拆分参考阈值。
- 单一职责：组件应有明确职责边界，复杂逻辑拆到 hooks 或 `lib/`。

## Children 与组合（NON-NEGOTIABLE）

参考 `code-modification-safety` 规则：

- 修改前必须确认父组件对 `children` 是否有类型约束（`cloneElement`、`React.Children.map` 等）。
- 不要随意移除"看似多余"的包装元素，存在即可能有依赖。
