# 样式规范

## 基础原则（NON-NEGOTIABLE）

- 主样式方案为 **Tailwind CSS**，配合 `lib/utils.ts` 中的 `cn` 组合动态 class。
- 原子组件来自 **shadcn**，位于 `components/ui/`，不得直接修改原子层视觉 token，扩展请在业务层封装。
- 复杂交互控件使用 **Ant Design**，并保持 Antd 与 Tailwind 的视觉一致。
- **禁止**使用全局 CSS 添加业务样式（仅 `app/globals.css` 维护重置与主题变量）。
- **禁止**新增 SCSS Modules / CSS Modules（`*.module.scss`、`*.module.css`）。

## 主题变量（NON-NEGOTIABLE）

- 颜色、间距、圆角通过 `tailwind.config.ts` 的 theme token 与 `app/globals.css` 的 CSS 变量定义。
- 暗色模式由 `next-themes` 切换 `<html class="dark">`，组件使用 `dark:` 前缀响应。
- **禁止**在组件内硬编码颜色值（`#xxx`、`rgb()`），必须使用 Tailwind 类或 CSS 变量。
- 使用 Antd 时通过 ConfigProvider 主题对齐，避免在组件内覆盖 Antd 内部样式。

## 动态 class 与组合

```tsx
import { cn } from '@/lib/utils';

<div className={cn('rounded-lg p-4 bg-card', isActive && 'ring-2 ring-primary')} />
```

- 同一组件内同时使用 Tailwind 与 Antd 时，避免互相覆盖：让 Antd 控件保持默认外观，Tailwind 只控制布局。

## 设计稿颜色提取

| 属性 | 含义 | 提取方式 |
|------|------|----------|
| `fill` | 背景色 | Tailwind 任意值 / CSS 变量 |
| `stroke` | 边框色 | `border-[color]` 或 token |
| `opacity` | 透明度 | `opacity-*` 或 RGBA |
| `background_blur` | 毛玻璃 | `backdrop-blur-*` |
| `shadow` | 阴影 | `shadow-*` 或 token |
| `cornerRadius` | 圆角 | `rounded-*` |

提取步骤：

1. 定位目标元素（按钮、卡片等）。
2. 查找 `fill` 属性获取背景色，转换为 Tailwind token；如无对应 token，再使用 `bg-[rgba(...)]` 任意值。
3. 检查 `effect`（模糊、阴影），落到 `backdrop-blur-*`、`shadow-*`。
4. 文字颜色取 `children.text` 节点，落到 `text-*` 类。

带透明度的颜色（如 `#13131c73`）转换为 `rgba(19,19,28,0.45)` 后再写入 Tailwind 任意值。
