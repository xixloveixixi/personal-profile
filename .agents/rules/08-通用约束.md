# 通用约束

## 语言规范（NON-NEGOTIABLE）

- 所有文档、代码注释、提交信息使用中文。
- 代码变量 / 函数命名使用英文，遵循 `02-编码规范.md`。

## Harness 必读（NON-NEGOTIABLE）

按 `AGENTS.md`，执行任何开发任务前必须先读：

- `docs/dev-harness/stage-plan.md`：当前阶段与 Gate。
- `docs/dev-harness/progress-log.md`：进度记录。
- `docs/dev-harness/pitfalls.md`：复用踩坑。

工作流程：判断阶段 → 检查 Gate → 推进一个可验证小闭环 → 更新 progress-log → 出现踩坑写入 pitfalls。

## 项目核心边界（NON-NEGOTIABLE）

- 博客内容只来自 Notion，**禁止**把博客迁出 Notion 或在 Go 后端建博客 CRUD。
- 公开区 Chat **禁止**读取私有学习数据。
- 私有学习数据查询必须按 `owner_id` 限定，禁止从外部传入 `userId`。
- Go 后端推进遵循 Harness Stage 1，**未确认接口与表结构前禁止写业务后端代码**。
- **禁止**一次性大范围重构现有前端。

## 可观测性与兜底

- AI 任务（Chat、计划生成、复盘总结）需要完整链路：`traceId`、消息引用、模型与工具调用日志。
- 失败时提供兜底提示与重试机制；流式接口必须能在中断时降级到非流式或友好错误。
- 关键 Route Handler 在异常时返回统一响应结构，参见 `05-API规范.md`。

## 占位元素使用规范

资源未确定时使用占位元素，不实现临时方案：

| 场景 | 处理方式 |
|------|----------|
| 图标 | 占位 div + Tailwind 尺寸类，附 TODO 注释 |
| 图片 | `next/image` 指向占位资源或占位 div |
| 后续替换 | 标记 `// TODO:` 注明替换条件 |

禁止：

- 用临时 SVG 顶替最终图标。
- 使用第三方 placeholder 图片服务。
- 留下未标记的占位代码。
