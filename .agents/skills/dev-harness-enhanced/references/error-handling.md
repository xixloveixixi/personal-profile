# 错误处理与降级

FSM 运行中遇到异常时的处理策略。

## 文件缺失

| 缺失文件 | 处理 |
|---------|------|
| stage-plan.md | 询问用户当前阶段，口头确认后继续 |
| api-contract.md | 阻断进入 CODING，提示先完成 DESIGN |
| schema.md | 同上 |
| progress-log.md | WRAP_UP 时自动创建 |

## 文件格式异常

| 场景 | 处理 |
|------|------|
| 契约文件无冻结标记 | 提示添加，不猜测内容 |
| stage-plan.md 无当前阶段标记 | 列出所有阶段，让用户选择 |
| Gate checkbox 解析失败 | 输出原始内容，让用户判断 |

## 门禁连续失败

| 场景 | 处理 |
|------|------|
| 软门禁 2 轮不收敛 | 升级为 HARD_FAIL，等待用户决策 |
| 硬门禁连续失败 2 次 | 停下，输出卡点报告，列出尝试过的修复 |
| 相同错误反复出现 | 提示可能是环境/依赖问题，建议用户介入 |

## 用户要求跳过

| 场景 | 处理 |
|------|------|
| 跳过 LEARNING | 允许，走快速通道 |
| 跳过 Gate 检查 | 说明风险，记录跳过事实到 progress-log.md，尊重决定 |
| 跳过 ACCEPTANCE | 不建议，但用户坚持则记录后进入 WRAP_UP |

## 阶段游标问题

| 场景 | 处理 |
|------|------|
| 游标未挪动 | 主动提醒，不代改 |
| 游标与实际进度不符 | 询问用户以哪个为准 |

## 降级原则

1. **不猜测** — 文件内容异常时展示原文，让用户判断
2. **不阻塞** — 非关键文件缺失时口头确认继续
3. **记录** — 所有跳过/降级决策记入 progress-log.md
4. **尊重** — 最终决定权在用户
