# 状态查询输出格式

## AI 执行步骤

1. 读取 `docs/dev-harness/stage-plan.md`
2. 找到 `## 当前阶段：` 开头的段落
3. 在该段落中，按 `#### Gate X：` 分节，统计每节的 `- [x]` 和 `- [ ]` 数量
4. 按下方格式输出

## 输出格式

```
当前阶段：[阶段名称]

Gate A (需求): [x/total]
Gate B (设计): [x/total]
Gate C (学习): [x/total] — 注：需用户自己勾选
Gate D (编码): [x/total]
Gate E (验收): [x/total]
Gate F (沉淀): [x/total]

下一步：[基于未完成的最早 Gate 给出建议]
```

## 解析规则

- Gate 标题格式：`#### Gate X：xxx闸门` 或 `#### Gate X: xxx`
- 勾选项格式：`- [x]` 表示完成，`- [ ]` 表示未完成
- 如果某个 Gate 段落不存在，该行输出 `未定义`
- 如果文件中没有 `## 当前阶段：` 前缀的标题，输出错误提示（见下）

## 失败降级

- 文件不存在：`stage-plan.md 不存在，请确认文件路径`
- 无当前阶段标记：`stage-plan.md 中未找到"## 当前阶段："标记，可能需要挪动游标`
- 格式异常：`stage-plan.md 格式异常，建议手动检查`
