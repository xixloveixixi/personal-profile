# 闭环产出物输出格式

每个编码闭环结束后，AI 必须输出以下内容。

## 输出格式

```
## 闭环完成

**文件变更**：
- 新增：[文件路径列表]
- 修改：[文件路径列表]
- 删除：[文件路径列表]

**构建验证**：
[实际 terminal 输出，不是"通过"两个字]

**Gate E 进度**：[已完成项/总项]
- [x] 已完成的检查项
- [ ] 未完成的检查项
```

## 规则

- 构建验证必须是 `go build ./...` 或 `npm run build` 的实际输出
- 不能用"编译通过"代替实际输出——贴前 3-5 行即可，证明确实跑了
- Gate E 进度对照 `stage-plan.md` 中当前阶段的 Gate E 清单
- 输出后等待用户确认，不自动推进下一个闭环
